Installing VoiceCode

To install VoiceCode, follow the following steps
  • 0. Install support software
  • 1. Unzip the files
  • 2. Set environment variables
    • 2.1 VCODE_HOME variable
    • 2.2 PYTHONPATH variable
  • 3. Create a VoiceCode user
  • 4. Delete pickled symbol dictionary
  • 5. Test the installation
  • Trouble shooting

0. Install support software

In order to use VoiceCode, you must first install the following support software.
  • Python 1.5 (Python 2.0 not supported for now)
  • NatLink
  • Dragon Naturally Speaking
  • wxPython 2.2
  • PyXML

1. Unzip the files

Unzip the VoiceCode files to a directory on your PC. Make sure you preserve subdirectory structure.

2. Set environment variables

Next, you must set the following environment variables.

2.1 VCODE_HOME variable

Set VCODE_HOME to the root of the directory where you unzipped the VoiceCode files.

2.2 PYTHONPATH variable

Add the following to your PYTHONPATH variable:

%VCODE_HOME%/Mediator;%VCODE_HOME%/Admin;C:/NatLink

Assuming of course you have installed the NatLink files under C:/NatLink.

3. Create a VoiceCode user

VoiceCode connects to your Speech Recognition system through a special user. This is to avoid biasing your regular user towards dictation of code (which tends to be very different from dictating English). Before using or testing VoiceCode, you must therefore create that special user. The user name should be VoiceCode. Instead of starting with a brand new user, we recommend that you start with a copy of a user that you have already trained to plain English (if you have one). To make a copy of a user, do the following:

  • open the user in NatSpeak
  • Select User->Backup from the menu.
  • Select User->Restore. This will create a duplicate of the user, with a name like "UserName-Restored1".
  • Select User->Open, highlight the newly created user, and press Rename. Rename it to "VoiceCode".
  • Select your actual user, and press Open. This is just because I don't trust NatSpeak not to undo the rename if you hit Cancel, and is probably paranoia.

4. Delete pickled symbol dictionary

If you are installing a new version of VoiceCode over an existing one, you should erase (or rename) the following file:

%VCODE_HOME%\Data\State\symdict.pkl

5. Test the installation

You may run a regression test to make sure that the installation succeeded. Simply do the following:

  • 1. Make sure NatSpeak is running
  • 2. Turn off the microphone
  • 3. Go to a DOS window (the "server" window) and type:
    • cd %VCODE_HOME%\Mediator
    • python tcp_server.py -t all > foo
  • 4. Go to an other DOS window (the "client" window)
    • cd %VCODE_HOME%\Mediator
    • python test_tcp_server.py
  • 5. Keep the microphone off until test has finished.
This will run the regression test in a Client-Server configuration where the VoiceCode server communicates with a test client through TCP/IP. If you are not interested in testing the Client-Server aspects of VoiceCode, you can run a faster regression test by replacing steps 4 and 5 by the following step:
  • Go to a DOS window
  • cd %VCODE_HOME%\Admin
  • python test.py all > foo
This test take about 30-50% faster than the Client-Server one (it took 7 mins on a Pentium IV).

When test has finished running, compare the output file foo to the file %VCODE_HOME%/Data/Benchmark/correct_results.dat. The two files should be identical, except for timing information (which depends on the speed of your machine). You can compare the files using the Windows fc command or with a windows version of the Unix diff command (like the one found in Cygwin).

Note that you can use other applications while the regression test is running. However if you press the Shift or Ctrl while in an other application, utterances generated by the regression test will end up going to that application instead of to the regression test. This may end up typing some undesirable text into the application or even invoking commands of that application. It may also invalidate the output of the regression test, since it will miss some of the utterances.

This behaviour is due to the fact that Shift and Ctrl force NatSpeak to respectively interpret an utterance as dictation or command. You can fix this situation by reconfiguring this key mapping to employ an infrequently used key combination (for example, Ctrl+Shift and Alt+Shift). To do this, go to NatSpeak and do:

  • click Tools->Options->Hotkeys
  • click Force command recognition
  • type key sequence you want to associate with that action (note that only certain key sequences are allowed)
  • do the same with Force dictation recognition
