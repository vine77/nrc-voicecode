Installing VoiceCode

To install VoiceCode, follow the following steps
  • 0. Install support software
  • 1. Unzip the files
  • 2. Set environment variables
    • 2.1 VCODE_HOME variable
    • 2.2 PYTHONPATH variable
    • 2.3 VCODE_NOSPEECH variable
  • Test the installation

0. Install support software

In order to use VoiceCode, you must first install the following support software.
  • Python
  • NatLink
  • Dragon Naturally Speaking
Note that you can still run VoiceCode in simulation mode without having to install Dragon Naturally Speaking. You must then set the VCODE_NOSPEECH environment variable.

Note however that NatLink must be installed on your machine whether or not you install NatSpeak. If you don't install NatSpeak, you can just copy NatLink to your machine without running EnableNL.

1. Unzip the files

Unzip the VoiceCode files to a directory on your PC. Make sure you preserve subdirectries structure.

2. Set environment variables

Next, you must set the following environment variables.

2.1 VCODE_HOME variable

Set VCODE_HOME to the root of the directory where you unzipped the VoiceCode files.

2.2 PYTHONPATH variable

Add the following to your PYTHONPATH variable:

%VCODE_HOME%/Mediator; %VCODE_HOME%/Admin; C:/NatLink Assuming of course you have installed the NatLink files under C:/NatLink.

2.3 VCODE_NOSPEECH variable

If you don't have installed Dragon Naturally Speaking, you can still run VoiceCode in simulation and use keyboard commands to simulate dictation events.

To do this you must define environment variable VCODE_NOSPEECH (the value you set it to is irrelevant).

Test the installation

To test the installation do the following: 

cd %VCODE_HOME%\Admin
python test.py all > foo
then compare the output file foo to the file %VCODE_HOME%/Data/Benchmark/correct_results.dat. The two files should be identical except for a few lines that contain parts of the %VCODE_HOME% path (those lines obviously will vary depending on where you installed VoiceCode) and lines that contain profiling information (those depend on how fast your computer is). You can compare the files using the Windows fc command or with a windows version of the Unix diff command (like the one found in Cygwin)
