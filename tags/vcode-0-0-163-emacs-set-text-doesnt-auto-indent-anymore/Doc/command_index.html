Language-independant commands

Punctuation marks

Code contains a lot of punctuation marks and therefore, VoiceCode defines easy to utter aliases for commonly used sequences of punctuation. When dictating code, programmers tend to use a mix of syntactic or semantic utterances for punctuation marks.

A syntactic utterance is one that is independant of the programming language in use. It doesn't convey any information about the semantics of the punctuation or sequence of punctuatiojn. For example, "pipe sign" or "vertical bar" are syntactic utterances for the punctuation mark '|'.

A semantic utterance is one that depends on the programming language in use. It is tied to the semantics of the punctuation mark in that programming language. For example in C, "single or" is a semantic utterances for '|'.

VoiceCode supports both syntactic and semantic utterances for punctuation marks. The table below lists all the syntactic utterances supported for punctuation marks. For a list of semantic utterance for a particular language, consult the language command index.

NavigateByPunctuation commands

VoiceCode supports a series of commands we call NavigateByPunctuation. The idea is that code has lots of punctuation, so why not use them as targets for navigation commands.

NavigateByPunctuation commands are not programming language dependant and are therefore available when dictating code in any language.

NavigateByPunctuation commands have the following general form:

"(before|after)* (next|previous)* <punctuationMark>"

Here, <punctuationMark> is a voice alias for a punctuation mark or a commonly used sequence of punctuation marks.

The other qualifiers in the command specify if:

  • the cursor should be moved to the previous (previous qualifier) or next (next qualifier) occurence of <punctuationMark>
  • the cursor should be put at the beginning (before qualifier) or after (after qualifier) the occurence

By default, unless specified otherwise by a qualifier, the command moves the cursor to:

  • the next occurence
  • at the end of that occurence

Note that in addition NavigateByPunctuation commands, VoiceCode supports special commands for typing balanced expressions and navigating through them.

Balanced expressions

VoiceCode has commands for inserting balanced expressions and jumping out of them. The commands for inserting balanced expressions have the general form:

"between <exprName>"
"<exprName> pair"
"empty <exprName>"

The "between <exprName>" form is used to type the balanced expression and put the cursor inside the expression. The "empty <exprName>" form types the balanced expression but puts the cursor right after it.

For example "between brackets" types '[^]' (where ^ denotes the cursor position) while "empty brackets" types '[]^'.

There are also commands for jumping out of balanced expressions. They have the generic form:

"jump out"
"back jump out"
"jump back out"
"out of <exprName>"
"back out of <exprName>"

The command "jump out" command moves forward out of the innermost balanced expression while "back jump out" and "jump back out" move backward.

The command "out of <exprName>" moves forward out of the innermost balanced expression of type <exprName>, while "out of <exprName>" moves backward.

The table below is a list of all the language independant values for <exprName>. For a list of possible values for a particular language, consult the command index for that language.

NavigateByPseudoCode commands

VoiceCode supports commands for moving the cursor to or selecting a piece of code that is visible on the screen. Contrarily to most VoiceCode command, these commands must be invoked as a single utterance, that is you must pause before and after the command, but not in the middle.

NavigateByPseudoCode commands have the general form:

"select <startCode> through <endCode>"
"(go|select)* (before|after)* (previous|next)* <pseudoCode>"

In the first form, <startCode> and <endCode> are pseudo-code that correspond to two pieces of native code that are visible on the screen. The command simply selects a piece of code that starts with an occurence of <startCode> and ends with an occurence of <endCode>.

In the second form, the <pseudoCode> argument is the pseudo-code form of the code that you want to move the cursor to or select.

The other qualifiers in the command are used to specify whether:

  • an occurence of <pseudoCode> should be selected ("select" qualifier) or if the cursor should just be moved to the occurence ("go" qualifier).
  • the cursor should be put before ("before" qualifier) or after ("after" qualifier) the occurence
  • the occurence of <pseudoCode> should be before ("previous" qualifier) or after ("next" qualifier) the current position of the cursor
Note that at least one of the qualifiers should be present. In other words, uttering just "<pseudoCode>" will not move the cursor to nor select an occurence of <pseudoCode>. It will just type it.

By default, unless there are qualifiers that specify otherwise the command just moves the cursor (i.e. no selection) after the occurence of <pseudoCode> that's closest to the current cursor position.

Command repetition

VoiceCode supports commands for repeating the last command that was uttered. These commands only apply if the last command was of type "repeatable". Examples of repeatable commands are PgUp, PgDown and search commands. If the previous command was not of type "repeatable", then the utterance is treated as a pseudo-symbol.

Commands for repeating the last command have the general form:

"again|do that again|repeat|repeat that"
"<N> times"
"repeat <N> times"
"again <N> times"

where <N> is a number between 1 and 10. All of those commands repeat the previous command (if it was "repeatable").

Note that the behaviour of "<N> times" is slightly different from the behaviour of "repeat <N> times" and "again <N> times".

Commands "repeat <N> times" and "again <N> times" always repeat the previous command (if it's "repeatable") exactly <N> times.

On the other hand, "<N> times" will repeat the previous command <N> times only if that previous command has already been repeated at least once. Otherwise, it will repeat it <N> - 1 times. The rationale for this is that if the user says:

"<someCommand> <N> times"

she probably means that <someCommand> should be executed a total of <N> times. But the command will be executed once by the utterance "<someCommand>", and therefore, "<N> times" should only repeat it <N> - 1 times. If on the other hand the user says:

"<someCommand> <M> times", followed by: "<N> times"

then she probably means that "<someCommand>" should be repeated <M> + <N> times. So the second utterance should repeat <someCommand> exactly <N> times.

Direction qualification

VoiceCode also comes with commands for qualifying the "direction" of the last command. Like the commands used for repetition, these repeat the last command. However, they only repeat it once and they may change its "direction". Those commands only apply if the last command was both "repeatable" and "bidirectionnal". Examples of such commands are: PgUp, PgDown and search commands. If the last command was not of type "repeatable" or "bidirectionnal", then the utterance is treated as a pseudo-symbol.

Commands for direction qualification have the generic form:

"reverse (direction)*"
"backward|upward|leftward|previous one"
"forward|downward|rightward|next one"

Command "reverse (direction)*" simply repeats the previous command in the opposite direction.

Command "backward|upward|leftward|previous one" repeats the previous command in the backward direction.

Command "forward|downward|rightward|next one" repeats the previous command in the forward direction.
